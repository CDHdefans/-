### 规则体系

| 设定 | 值 | 说明 |
| --- | --- | --- |
| 山的底层定位 | 山不害人，山只是有边界 | 禁止把山写成反派、怪物或审判者 |
| 大山发脾气 | 人物口语，机制层为规矩闭合反应 | 不是神罚；是坏规矩后边界开始闭合 |
| 意图回响的机制 | 坏规矩后的感知偏差，不是审判 | 禁止写成"山神惩罚善恶" |
| 规则触发四类 | 地形触发、行为触发、关系触发、记录触发 | 规则从真实户外流程污染中显形 |
| 规矩不能外说 | 不能把规矩整理成公开攻略、怪谈手册或可复现样本 | 正文只能让角色把它落成动作：停、绕、等、删、别回应、不公开坐标 |
| 灵异发生的四个原因 | 自然规矩被坏、救援介入受牵连、旧案未闭合、记录过度与复现伤害 | 每个灵异点都要先追溯山的边界为何显形；白线不是灵异发动机，但会把观察点支线误写成安全路线 |
| 三层异常的顺序 | 错位 → 替身 → 召回 | 第一卷三层均有出现，但高等级出现须有低等级铺垫 |
| 召回层本质 | 边界闭合，不是主动猎杀 | 可伤人，但不是为了伤人 |
| 规则完全可被破解吗 | 否 | 规则不能被彻底掌握和利用 |
| 正文叙事口径 | 不出现章节号、阶段名和作者回看 | 正文只能写角色能感知的前情；“第几章/第几阶段/前几阶段”只允许在大纲、进度、标题和写作备注中使用 |

### 白线计划

| 设定 | 值 | 说明 |
| --- | --- | --- |
| 白线计划定位 | 有救人信念、有项目利益、有遮掩动机的高山边界安全网络 | 成员不自认反派，公开目标是真实的；旧案中它也是事故条件制造者之一 |
| 第一卷白线计划的存在感 | 只露尾巴 | 环境采样设备、档案编号、缓存标签、训练样本裁剪、路线包来源，不能正面登场 |
| 白线计划越记录的后果 | 特定天气和本地经验下才可能通行的观察点支线，会被误写成稳定安全样本 | 他们以为在记录救人，实际在把“有限条件可走”包装成“普遍可走” |
| 白线计划的根本错误 | 不是创造规则或为了害人而害人，而是观察点审核不足、天气窗口误判、样本外推、资料包外溢，并为了项目利益遮掩旧案 | 不能让白线替代山成为最终怪物，也不能写成无辜误伤 |
| 白线观测队口径 | 旧案中的另一支被救队伍就是白线观测队 | 他们被旧队救出后因保密协议、项目压力或补偿沉默；不能再改成随机游客或邪恶诱饵 |
| 第一卷白线露出方式 | 资料链模型化，不是实时篡改 | 禁止把白线写成当场改文件名、短信、设备名的黑客；只能通过旧档案、缓存索引、转写标签、样本裁剪、路线包来源和设备残留露尾巴 |
| 白线成员红线 | 不自称掌控规则，不为了害人而害人 | 每个白线线索都要能回答：这件事在他们逻辑里如何救人 |
| 白线与旧队认知 | 白线和学校前期不知道旧队仍活在第六天线里 | 他们以为事故已经结束或永远无法回归，后期才逐渐意识到程野在接近失控真相 |
